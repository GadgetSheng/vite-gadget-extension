SOFT 方式 VibeCoding
### 第一步：梳理需求，将“许愿”变成“施工图”​
视频指出，许多人失败的第一步在于用模糊的想法直接指挥AI，导致代码混乱。正确的做法是使用一个“需求建模公式”，将想法转化为结构化的文档。例如，将“我想要一个从照片提取配色灵感的工具”分解为：需求概述、核心功能（上传图片、分析色调、显示/复制颜色代码等）、交互流程、技术要求。这一步创建了一个清晰的“PRD”（产品需求文档），为AI提供了明确的指令基础。
### 第二步：完善产品文档，与AI确认细节
有了初步框架后，不要立刻让AI写代码。而是将PRD文档提供给AI（如Cursor），并给出关键指令：“先不要开发，帮我将这个想法补充成完整详细的产品文档，并告诉我关键部分的实现方案，如有问题请和我确认。”AI会阅读并完善文档，同时提出需要确认的关键问题（如色卡保存格式、颜色代码规格等）。通过回答这些问题，你与AI就产品细节达成共识，形成一份双方确认的详细蓝图。
### 第三步：制作Demo，前端先行验证产品形态
为了避免开发完成后才发现产品与想象不符，可以采用“前端先行”策略。即让AI先根据产品文档创建一个仅包含页面布局和静态展示数据的Demo，不实现复杂逻辑。这样可以最低成本、最直观地预览最终产品的界面和交互，确保大方向正确后再投入开发。
### 第四步：建立开发计划，利用Plan模式避免AI“失忆”​
当真正开始编码时，项目文件会迅速增多，可能导致AI因上下文被占满而“失忆”或乱写代码。解决方案是使用AI工具的“Plan”（计划）模式。指示AI根据已有的文档和Demo制定一个分阶段的详细开发计划，并将其保存到工作区。然后使用“Build”按钮，AI会自动将计划拆解成待办事项，并有序地逐个完成，从而保持上下文的清晰和项目的可控性。
### 第五步：验收与迭代，使用ACE反馈法驱动优化
开发完成后，需要进行测试和验收。如果发现功能或交互上的问题，视频推荐使用“ACE反馈法”与AI沟通：A（截图）、C（位置）、E（问题）、E（预期）。具体来说，就是对问题区域截图，然后清晰描述问题发生的位置、当前的具体问题，以及你期望的改进结果。例如，反馈“色卡部分的复制功能需要精准点击特定文字，操作不直观，期望在每个色值类型后增加显性的复制按钮”。AI会根据这种结构化反馈进行有效迭代。
### 总结
整个流程通过“梳理需求 → 完善文档 → 制作Demo → 制定计划 → 开发验收”这五个步骤，将AI编程从一个依赖运气的“黑箱”过程，转变为有方法、可控制的产品开发流程。它强调的核心是：清晰的输入（结构化需求）是获得优质输出（可用代码）的前提。掌握了这套方法，即使没有编程基础，也能通过指挥AI工具来解决实际问题，提升个人生产力。
